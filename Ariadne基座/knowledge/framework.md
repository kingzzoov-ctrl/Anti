# Ariadne 择偶心理分析框架

> 本文档是 Ariadne 系统的理论基石，为问询 Skill 和报告 Skill 提供统一的分析语言与推理依据。

---

## 一、核心理念：表层偏好 vs 深层需求

### 1.1 定义

| 概念 | 含义 | 典型表述 |
|------|------|----------|
| **表层偏好** | 用户能够自主表达、有意识认同的择偶条件 | "我想找一个身高180以上的男生" |
| **深层需求** | 驱动偏好形成的底层心理动力，用户往往未清晰觉察 | 对安全感和被保护的渴望 |

### 1.2 偏好与需求的关系

- **一致型**：偏好直接指向真实需求（偏好"温柔的人" ← 需求"被接纳和包容"）
- **偏移型**：偏好只部分映射需求（偏好"有钱的人" ← 需求"生活稳定感" + 未被表达的"被重视感"）
- **矛盾型**：偏好与需求存在冲突（偏好"独立不粘人的人" ← 实际需求"高度亲密连接"但害怕被拒绝）
- **补偿型**：偏好源自对过往缺失的反向补偿（原生家庭缺乏关爱 → 偏好"特别体贴的人"，但实际需求是"学会自我关爱"）

### 1.3 常见表层-深层映射模式

| 表层偏好 | 可能的深层需求 | 分析线索 |
|----------|---------------|----------|
| 要求对方经济条件好 | 安全感、被照顾的渴望、对不确定性的恐惧 | 成长环境是否经历过经济不稳定 |
| 要求对方外貌出众 | 社会认同感、自我价值确认 | 是否在意他人评价、社交中是否缺乏自信 |
| 要求对方幽默风趣 | 情感连接的渴望、回避深层沟通的替代策略 | 过往关系中是否有"感觉聊不到一起"的经历 |
| 要求对方独立不粘人 | 对亲密关系的恐惧、害怕失控、回避型依恋 | 过去关系中是否有"被缠住"的窒息感 |
| 要求对方事业心强 | 对上进的认同、自身价值观映射、被仰望的需要 | 是否自身事业心强、是否对"平庸"有焦虑 |
| 要求对方顾家体贴 | 被关注、被优先选择的渴望 | 原生家庭中是否感到被忽视 |
| 要求对方兴趣广泛 | 对无聊关系的恐惧、自身缺乏安全感需要外部刺激 | 独处时是否感到空虚 |
| 不介意任何条件/"随缘" | 低自我价值感、害怕被拒绝所以不敢提要求 | 是否在其他领域也难以表达自己的需求 |

## 二、依恋理论（Attachment Theory）

### 2.1 四种依恋类型

**安全型（Secure）**
- 特征：对亲密关系感到自在，能够信任伴侣，能健康地表达需求
- 在择偶中的表现：偏好开放、清晰，能区分"想要"和"需要"
- 信号词："我觉得关系是相互的"、"我可以接受对方有自己的空间"

**焦虑型（Anxious-Preoccupied）**
- 特征：渴望极度亲密，害怕被抛弃，对关系中的信号过度敏感
- 在择偶中的表现：强调"在乎我"、"秒回消息"、"时刻想着我"等即时回应
- 信号词："我最受不了已读不回"、"我需要确定他是真的喜欢我"
- 深层需求：自我价值感的外部确认

**回避型（Dismissive-Avoidant）**
- 特征：重视独立性，对亲密感到不适，倾向于压抑情感需求
- 在择偶中的表现：强调"各自有空间"、"不要太粘人"、"理性一点"
- 信号词："我觉得两个人不需要天天在一起"、"我比较独立"
- 深层需求：安全的亲密连接（但因害怕受伤而防御）

**混乱型（Fearful-Avoidant）**
- 特征：同时渴望亲密又害怕亲密，在接近和退缩间摇摆
- 在择偶中的表现：要求矛盾（既要体贴又要给空间、既要热烈又怕受伤）
- 信号词："我不知道自己到底想要什么样的人"、"每段感情到一定程度我就想逃"
- 深层需求：安全感 + 自主感的平衡

### 2.2 依恋类型的问询识别策略

| 依恋类型 | 关键问询方向 | 典型回答模式 |
|----------|-------------|-------------|
| 安全型 | 描述理想关系状态 | 平衡、具体、不极端 |
| 焦虑型 | 伴侣没及时回消息时的反应 | 焦虑、反复确认、灾难化想象 |
| 回避型 | 对方想要更多亲密时的感受 | 不适、需要空间、觉得对方"太粘" |
| 混乱型 | 过去几段感情的结束原因 | 模式矛盾、忽近忽远、自我否定 |

## 三、关系需求层次模型

借鉴马斯洛需求层次理论，建立亲密关系场景下的五层需求模型：

```
┌─────────────────────┐
│    自我实现需求       │ → 关系中成为更好的自己，共同成长
│   （Growth）         │
├─────────────────────┤
│    尊重需求          │ → 被理解、被欣赏、平等对待
│   （Esteem）         │
├─────────────────────┤
│    归属需求          │ → 情感连接、被爱、被选择
│   （Belonging）      │
├─────────────────────┤
│    安全需求          │ → 关系稳定、忠诚、经济安全
│   （Safety）         │
├─────────────────────┤
│    生理/基础需求      │ → 身体吸引力、性和谐、生活习惯兼容
│   （Basic）          │
└─────────────────────┘
```

### 3.1 分析应用

- 用户表述的偏好大多集中在某一两层，而非均匀分布
- 过度强调某一层往往意味着该层需求长期未被满足
- 完全不提及某一层可能是刻意回避或尚未觉察
- 健康的择偶需求应该在多个层次上都有合理期待

### 3.2 问询中的层次识别

| 需求层次 | 用户可能的表述 | 分析要点 |
|----------|--------------|----------|
| 基础 | "长得顺眼就行"、"生活习惯别太离谱" | 是否因为不敢要求更多而降低标准 |
| 安全 | "有房有车"、"工作稳定"、"人品好不花心" | 是否过度聚焦安全感而忽略情感连接 |
| 归属 | "两个人聊得来"、"有共同语言"、"被爱的感觉" | 核心需求，大部分人最渴望但表达方式各异 |
| 尊重 | "尊重我的工作"、"不要PUA"、"三观一致" | 是否有过被贬低/不被理解的经历 |
| 成长 | "能一起进步"、"互相成就" | 是否对现状不满、将关系视为改变契机 |

## 四、常见择偶认知偏差

### 4.1 偏差清单

| 偏差名称 | 描述 | 问询中如何识别 |
|----------|------|--------------|
| **光环效应** | 因某一突出特质而美化整个人 | "他很帅所以我觉得其他方面应该也不错" |
| **补偿心理** | 试图通过伴侣弥补自身或原生家庭的缺失 | 对某类特质的执着与个人成长经历高度关联 |
| **投射效应** | 将自己的特质/期望投射到理想伴侣上 | 理想伴侣描述与自我描述高度重合或完全相反 |
| **稀缺偏见** | 认为符合条件的人极少，产生焦虑或妥协 | "像这样的人太少了"、年龄焦虑 |
| **沉没成本** | 因过往投入而不愿调整择偶标准 | "我都等了这么久了"、拒绝降低已不合理的标准 |
| **社会比较** | 以他人的关系/伴侣为标准 | "我朋友老公就是..."、"别人都..." |
| **理想化偏差** | 构建一个完美但不存在的伴侣形象 | 条件清单过长且互相矛盾 |
| **否认偏差** | 声称"没什么要求"实则压抑了真实需求 | 回答过于简短、回避具体描述 |
| **近因效应** | 因最近一段经历过度调整择偶标准 | 新的"必须"条件与最近一段感情的痛点高度对应 |

### 4.2 偏差分析原则

- 识别偏差不是为了"纠正"用户，而是帮助用户看到更完整的自己
- 偏差本身不是错误，而是心理保护机制的外化
- 报告中应以共情方式呈现偏差分析，避免评判性语言

## 五、价值观匹配维度

### 5.1 核心价值观轴

| 维度 | 轴的两端 | 对关系的影响 |
|------|---------|-------------|
| **个体-集体** | 重视个人自由 ↔ 重视家庭/集体 | 决定关系中的独立性与融合度 |
| **稳定-冒险** | 追求安定 ↔ 追求新鲜刺激 | 影响生活方式和重大决策 |
| **理性-感性** | 逻辑优先 ↔ 情感优先 | 决定冲突解决方式和沟通模式 |
| **成就-体验** | 目标驱动 ↔ 过程享受 | 影响对"成功"和"幸福"的定义 |
| **传统-开放** | 遵循传统 ↔ 拥抱变化 | 影响性别角色、家庭分工等核心议题 |

### 5.2 价值观匹配原则

- **核心价值观**必须高度一致（通常 1-2 个最重要的维度）
- **次要价值观**允许差异但需要相互尊重
- 用户往往不会直接表述价值观，需要从具体选择和偏好中推断
- 价值观冲突是长期关系中最难调和的矛盾源

## 六、荣格认知功能（八维）

### 6.1 认知功能概述

荣格认知功能理论将人的心理过程分为 **8 种认知功能**，每个人都有偏好的使用顺序。这不是 MBTI 的四字母标签，而是更底层的认知偏好分析工具。在择偶场景中，认知功能匹配度直接影响**沟通方式、决策风格和冲突模式**。

### 6.2 八种认知功能

| 功能 | 简称 | 描述 | 在关系中的表现 |
|------|------|------|---------------|
| **外倾直觉** | Ne | 发散联想，看到多种可能性 | 喜欢跳跃式聊天，不断探索新话题和新体验 |
| **内倾直觉** | Ni | 深层洞察，追求本质和长远意义 | 需要伴侣理解自己的"直觉"，有时难以解释自己的感受来源 |
| **外倾感觉** | Se | 关注当下体验，享受感官刺激 | 重视实际行动和共同体验，约会风格偏活动型 |
| **内倾感觉** | Si | 重视经验记忆，关注细节和传统 | 珍视共同回忆，需要稳定和可预期的关系节奏 |
| **外倾思维** | Te | 效率导向，追求逻辑和系统 | 习惯用"解决问题"的方式回应伴侣情绪 |
| **内倾思维** | Ti | 内在逻辑分析，追求精确理解 | 需要时间独立思考，可能在情感表达上显得慢热 |
| **外倾情感** | Fe | 关注群体和谐，善于感知他人情绪 | 天然的关系维护者，可能为了和谐压抑自身需求 |
| **内倾情感** | Fi | 深层个人价值判断，重视真实性 | 对关系中的"假"极度敏感，需要深层次的价值观共鸣 |

### 6.3 认知功能对择偶的影响

**沟通匹配**：
- Ne-Ni：需要一个能跟上/理解自己思维深度或广度的人
- Se-Si：对"怎么过日子"的偏好差异巨大，需要识别
- 用户如果强调"聊得来"，核心可能是认知功能层面的共振

**决策风格匹配**：
- Te-Ti 偏好者：可能在情感表达上需要学习，择偶时易忽视情感连接的重要性
- Fe-Fi 偏好者：可能在关系中过度关注情感层面，忽视现实条件的适配

**冲突模式**：
- T 型偏好遇到 F 型偏好：一方要"讲道理"，一方要"被理解"
- J 功能（Te/Fe）与 P 功能（Ti/Fi）的节奏差异：一方要快速达成结论，一方需要时间内化

### 6.4 问询中的认知功能识别策略

**不直接问 MBTI 类型**（用户可能不了解或误判），而是通过行为线索推断：

| 认知功能 | 行为线索 | 关键问询方向 |
|----------|---------|-------------|
| Ne vs Ni | 描述未来时发散（Ne）还是聚焦（Ni） | "你想象未来的生活，脑子里是一幅清晰的画面，还是很多种可能？" |
| Se vs Si | 回忆往事时关注感官细节（Se）还是情感氛围（Si） | 观察用户叙述方式即可 |
| Te vs Ti | 面对问题是直接行动（Te）还是先分析（Ti） | "遇到跟伴侣的分歧，你第一反应是什么？" |
| Fe vs Fi | 关注他人感受（Fe）还是自身价值判断（Fi） | "朋友来找你诉苦，你会怎么做？" |

### 6.5 分析应用原则

- **不贴标签**：不对用户说"你是 INFJ"，而是说"你在做决定时倾向于先内心感受再理性分析"
- **用于理解而非筛选**：认知功能不决定"谁跟谁配"，但帮助理解"我们的沟通方式可能在哪些地方需要磨合"
- **功能栈互补 vs 共振**：相似的功能栈带来默契，互补的功能栈带来成长，没有绝对优劣
- **动态而非静态**：人的认知功能使用会随年龄和经历发展，分析关注"当前偏好"

## 七、关系定位模型：爱人、搭子与队友

### 7.1 核心理念

在所有具体择偶条件之前，有一个更根本的问题：**你找的到底是什么角色的人？**

这个定位直接决定了择偶标准的权重分配、关系期待的合理性，以及匹配策略的方向。很多人择偶困难的根源不是"条件不对"，而是**对关系本身的定位模糊或内在矛盾**。

### 7.2 三种关系定位

| 定位 | 核心诉求 | 关键特征 | 典型表述 |
|------|---------|---------|---------|
| **爱人** | 深度情感连接与灵魂共鸣 | 重视心动感、精神契合、情感深度 | "我想找一个让我心动的人"、"灵魂伴侣" |
| **搭子** | 生活陪伴与共享体验 | 重视相处舒适、兴趣匹配、日常默契 | "找个能一起玩的人就好"、"搭伙过日子" |
| **队友** | 共同目标与协作共赢 | 重视能力互补、三观一致、共同规划 | "一起奋斗"、"互相成就"、"合伙人" |

### 7.3 定位的复杂性

**大多数人的真实需求是混合型**，但各维度的权重不同：

- **爱人为主 + 队友为辅**：想要心动，但也在意对方的实力和方向一致性
- **队友为主 + 搭子为辅**：核心要目标一致，但希望日常相处轻松
- **搭子为主 + 爱人为辅**：主要要陪伴舒适，偶尔也渴望深层心动

**常见的定位矛盾**：

| 矛盾模式 | 描述 | 表现 |
|----------|------|------|
| **嘴上找爱人，实际要队友** | 口头强调心动，但条件清单全是硬件 | 每次"心动"后因现实条件不匹配而放弃 |
| **嘴上找队友，内心要爱人** | 理性地列条件匹配，但总觉得"少了点什么" | 条件完美的人面前始终燃不起激情 |
| **被社会期待推向队友，本心想要搭子** | 周围人都在"向上找"，但自己其实只想舒服 | 焦虑感强但行动力弱，约会疲惫 |
| **因受伤从爱人退缩到搭子** | 经历过深度情感创伤，用"差不多就行"自我保护 | 对深层亲密有渴望但有恐惧 |

### 7.4 定位与分析框架的关系

不同的关系定位，决定了分析框架的权重调整：

| 分析维度 | 爱人定位 | 搭子定位 | 队友定位 |
|----------|---------|---------|---------|
| **依恋理论** | 核心权重 | 中等权重 | 较低权重 |
| **认知功能** | 重要（共振感） | 重要（默契感） | 中等 |
| **价值观** | 深层一致性 | 大方向一致即可 | 核心权重 |
| **生活方式** | 中等 | 核心权重 | 中等 |
| **经济/事业** | 较低权重 | 中等 | 核心权重 |
| **原生家庭** | 重要 | 中等 | 较低 |

### 7.5 问询中的定位识别

**不直接问"你找爱人还是搭子还是队友"**（用户可能不理解或给出社会期望答案），而是通过以下策略推断：

1. **理想伴侣第一反应**：让用户快速描述理想伴侣时，最先提到的维度暴露定位
2. **关系中最快乐的时刻**：爱人型回忆情感瞬间，搭子型回忆共同体验，队友型回忆共同成就
3. **选择困境测试**：
   - "一个让你心动但生活习惯差异大的人 vs 一个相处舒服但没什么心动的人"（爱人 vs 搭子）
   - "一个跟你方向完全一致的强强联合 vs 一个平凡但让你特别放松的人"（队友 vs 搭子）
   - "一个灵魂契合但经济条件一般的人 vs 一个能力很强但情感表达不多的人"（爱人 vs 队友）
4. **分手/遗憾原因**：最大的遗憾指向最核心的定位

### 7.6 分析输出要求

- 报告中必须明确用户的**主关系定位**和**辅关系定位**
- 识别定位是否存在**内在矛盾**或**外在干扰**（社会期待、家庭压力等）
- 后续所有分析（偏好解读、建议等）都应在关系定位的语境下展开
- 定位本身没有对错，但**清晰的定位是有效择偶的前提**

## 八、分析输出规范

### 8.1 分析层次

每个维度的分析应包含三层：

1. **事实层**：用户明确表述了什么（直接引用）
2. **推断层**：基于事实合理推断的深层需求（标注推断依据）
3. **洞察层**：偏好与需求之间的关系模式（一致/偏移/矛盾/补偿）

### 8.2 语言规范

- 使用"可能"、"或许"、"这暗示着"等非断定性语言
- 始终保持尊重和共情的基调
- 避免病理化表述（不说"你有回避型依恋障碍"，说"你在亲密关系中可能更倾向于保持一定的独立空间"）
- 洞察应具有建设性，指向成长方向而非问题标签

### 8.3 置信度标注

对每个深层需求推断标注置信度：

- **高置信**：多条独立证据相互印证
- **中置信**：有合理依据但证据有限
- **低置信**：基于有限信息的探索性推断，建议进一步探索

## 九、伦理边界

1. **不做心理诊断**：系统提供择偶需求洞察，不是心理治疗工具
2. **不做价值判断**：没有"好的"或"坏的"择偶偏好，只有是否与真实需求一致
3. **尊重用户自主性**：洞察是供参考的镜子，最终决策权在用户
4. **隐私保护**：问询中获取的个人信息仅用于分析，提醒用户注意隐私
5. **积极导向**：每一条分析最终都应指向积极的、建设性的方向
