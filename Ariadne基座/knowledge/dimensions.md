# 问询维度与问题库

> 本文档定义了问询阶段的各维度及其对应问题库。问询 Skill 在对话中根据当前阶段和信息缺口，灵活选取和改编这些问题。
> 
> **使用原则**：问题不应逐条机械提问，而是作为话题引子和追问方向的参考。好的问询像一次深入的朋友聊天，而非结构化面试。

---

## 维度零：个人基本信息

### 核心目标
采集用户的**客观硬件条件**和**生活现状**。这些信息本身不是分析重点，但为后续所有维度提供**现实锚点**——脱离现实条件的深层分析是空中楼阁。

### 核心信息项

| 信息项 | 为什么需要 | 采集方式 |
|--------|----------|---------|
| **年龄** | 影响择偶市场现实、人生阶段和时间压力感知 | 直接询问或确认区间 |
| **学历** | 影响社交圈层、认知匹配预期和自我定位 | 自然聊到教育经历时采集 |
| **职业/工作** | 反映能力方向、时间精力分配和社会角色 | 聊现状时自然引出 |
| **事业阶段与计划** | 决定了关系在人生优先级中的位置 | "你现在工作到什么阶段了？接下来有什么打算？" |
| **经济水平** | 影响生活方式匹配和安全感基线 | 不直接问收入，通过生活方式间接判断 |
| **外形自我认知** | 影响自信度和择偶标准校准 | "你觉得自己属于什么类型？" |
| **定居城市/意向** | 现实约束中最硬的一条 | "你现在在哪个城市？有换城市的打算吗？" |
| **家庭背景概况** | 社会阶层、家庭期待的现实基础 | 后续原生家庭维度深入 |

### 采集原则

- **破冰期自然融入**：不要一上来就"报户口"，而是通过聊契机、现状、对生活的满意度等自然获取
- **不评判硬件条件**：无论用户条件如何，保持中性和尊重
- **注意经济话题的敏感性**：通过"你对现在的生活状态满意吗？""最满意的部分和想改变的部分？"间接了解
- **外形不需要精确描述**：关注的是用户**对自身外形的认知和接纳度**，而非客观评价
- **定居意向是硬约束**：必须明确，这会直接影响择偶范围的现实可行性

### 与深层分析的关联

| 基本信息 | 可能的深层分析线索 |
|----------|-----------------|
| 年龄较大但从未恋爱 | 可能有深层回避倾向或过高标准 |
| 事业上升期 | 关系可能被放在次优先级，"队友型"定位概率较高 |
| 经济压力大 | 择偶条件中的经济要求可能既是现实需求也是安全感补偿 |
| 对外形不自信 | 可能影响择偶标准的校准（过高补偿或过低自我限制） |
| 定居城市不确定 | 可能反映人生方向本身的不确定性 |
| 高学历/高收入 | 择偶池客观缩小，可能面临"向上匹配"困境 |

## 维度〇·五：关系定位 —— 爱人、搭子还是队友？

### 核心目标
在所有具体条件之前，先厘清用户**对关系本身的根本定位**。这是整个分析的方向盘——定位不同，同一个条件的权重和解读完全不同。

### 核心问题

1. **如果用三个词描述你理想中的关系，你会用哪三个词？**
   - 目的：第一反应暴露核心定位——情感词（心动、懂我、浪漫）→ 爱人；体验词（舒服、开心、默契）→ 搭子；目标词（一起、成长、稳定）→ 队友
   - 追问方向：这三个词的优先级是什么？

2. **你觉得一段好的感情，最核心的东西是什么？**
   - 目的：直接定位核心诉求
   - 追问方向：这个标准从何而来？

3. **选择困境（情境题，选择最接近的）**：
   - "一个让你特别心动但生活习惯差异很大的人 vs 一个相处很舒服但没什么心动感的人，你更倾向哪个？"
   - "一个在事业上能跟你强强联合的人 vs 一个可能平凡但让你特别放松的人？"
   - 目的：在矛盾情境中迫出真实偏好

4. **回忆过去的感情（或想象中的关系），让你最遗憾/最渴望的是什么？**
   - 目的：最大的缺失指向最核心的定位

### 矛盾检测线索
- 口头说"心动最重要"但条件清单全是硬件 → 嘴上爱人，实际队友
- 理性列了很多匹配条件但总觉得"缺点什么" → 被压抑的爱人需求
- 说"差不多就行"但对过去的深度情感经历有明显眷恋 → 受伤后从爱人退缩
- 定位受到强烈外部影响（父母催婚→被动变成队友型；朋友都在恋爱→被动变成搭子型）

## 维度一：基本情况与自我认知

### 核心目标
了解用户的基本背景，更重要的是了解 **用户如何看待自己**——自我认知的准确度直接影响择偶偏好的合理性。

### 核心问题

1. **你怎么形容自己？如果让你的好朋友来形容你，他们会怎么说？**
   - 目的：对比自我认知与他人视角，识别自我认知偏差
   - 追问方向：如果两者有差异，探索差异来源

2. **你觉得自己身上最吸引人的特质是什么？最希望改变的地方呢？**
   - 目的：了解自我接纳度和自我价值感
   - 追问方向：希望改变的地方是否影响了择偶标准

3. **你现在的生活状态满意吗？最满意的部分和最想改变的部分是？**
   - 目的：识别用户是否将择偶作为"解决现有问题"的途径
   - 矛盾线索：如果对生活极度不满但择偶条件很高，可能存在补偿心理

4. **你一般怎么度过周末？一个人的时候喜欢做什么？**
   - 目的：了解真实生活方式（而非理想化描述）

### 情境假设题

- "如果有一个完全了解你的人来评价你的择偶条件，你觉得他会说什么？"
- "如果你的条件翻倍（更有钱/更好看/更年轻），你的择偶标准会怎么变？"

### 矛盾检测线索
- 自我评价过高或过低 → 可能影响择偶标准的合理校准
- 对自身缺点的回避 → 可能存在投射效应（要求伴侣具备自己缺少的特质）

## 维度二：感情经历与模式

### 核心目标
通过过往经历提取 **关系行为模式**，这是预测深层需求最可靠的数据源。

### 核心问题

1. **能说说你过去的感情经历吗？印象最深的是哪一段？**
   - 目的：开放式引入，观察用户选择讲述哪段经历（选择本身就是信号）
   - 追问方向：为什么这段印象最深？是因为幸福还是痛苦？

2. **你过去的感情大多是怎么开始的？又是怎么结束的？**
   - 目的：识别关系启动模式和关系破裂模式
   - 追问方向：结束时你是哪一方？你觉得真正的原因是什么？

3. **在过去的关系中，你最享受的时刻是什么？最难受的时刻呢？**
   - 目的：直接定位情感需求的核心——什么让你快乐/痛苦
   - 追问方向：那个最难受的时刻，你是怎么处理的？

4. **你有没有发现自己在感情里有某种重复的模式？**
   - 目的：考察自我觉察程度
   - 追问方向：如果有，你觉得这个模式是怎么形成的？

5. **如果过去的某段感情能重来，你会怎么做不同的选择？**
   - 目的：了解用户从经历中学到了什么，以及哪些遗憾在影响当前择偶标准

### 情境假设题

- "假设你的前任（或最在意的那位）现在坐在你对面，你最想对他/她说什么？"
- "如果你最好的朋友跟你有一模一样的感情经历，你会给他/她什么建议？"

### 矛盾检测线索
- 说"不在意前任了"但叙述时情绪明显 → 未完成的情感议题可能影响择偶
- 所有分手都归因于对方 → 可能缺乏自我反思，需要温和引导
- 多段感情结束原因雷同 → 核心模式信号，高价值分析素材
- "我没怎么谈过恋爱"但年龄较大 → 可能有回避型依恋倾向或深层恐惧

## 维度三：原生家庭与依恋风格

### 核心目标
了解 **早期关系模板** 如何塑造了用户的亲密关系模式和择偶偏好。

### 核心问题

1. **你和父母的关系怎么样？小时候和现在有什么不同吗？**
   - 目的：了解依恋类型的形成背景
   - 追问方向：小时候遇到困难时，你会找谁？

2. **你父母之间的关系是什么样的？你觉得他们的相处模式对你有什么影响？**
   - 目的：原生家庭关系模板是择偶模式的最大影响因子
   - 追问方向：你希望自己的关系像他们一样吗？哪些地方一样，哪些不一样？

3. **小时候你觉得最被爱的时刻是什么？最孤独的时刻呢？**
   - 目的：定位核心情感需求的早期来源
   - 追问方向：这些感受现在还会出现吗？什么时候？

4. **家里人对你找对象这件事是什么态度？他们有什么期望？**
   - 目的：区分自我需求和家庭投射的需求
   - 矛盾线索：用户的偏好是否大量来自家庭期望而非内心

### 情境假设题

- "如果让你选择在一个理想的家庭中重新长大，那个家庭是什么样的？"
- "你觉得自己在关系中更像爸爸还是妈妈？"

### 矛盾检测线索
- 说"原生家庭挺好的"但具体描述中有明显缺失 → 理想化防御
- 择偶偏好与父母特质高度相似或完全相反 → 都可能是原生家庭模式的投射
- 对父母关系评价过于极端（完美/糟糕） → 缺乏中间地带的整合

## 维度四：生活方式与日常偏好

### 核心目标
了解用户 **真实的日常生活节奏和偏好**，这些"小事"往往是长期关系成败的关键。

### 核心问题

1. **你理想中的一天是什么样的？从早到晚。**
   - 目的：了解生活节奏和优先级排序
   - 追问方向：这样的一天多久能实现一次？差距在哪？

2. **你的社交圈是什么样的？你更喜欢一群人还是几个知心朋友？**
   - 目的：社交偏好反映内外向和关系深度偏好
   - 追问方向：你希望伴侣的社交风格跟你一样吗？

3. **在花钱这件事上，你是什么风格？有没有觉得"这个钱花得很值"的经历？**
   - 目的：金钱观是核心价值观的外显，也是关系中高频冲突源
   - 追问方向：你能接受伴侣在花钱上跟你风格很不一样吗？

4. **你对"个人空间"的需求是什么程度的？在关系中你能接受多大程度的融合？**
   - 目的：直接关联依恋类型和亲密关系舒适度

5. **你怎么处理压力和负面情绪？你希望伴侣在你难过的时候怎么做？**
   - 目的：情绪调节方式和对伴侣的期待，直接反映关系中的核心需求

### 情境假设题

- "如果下班回到家特别累，你希望伴侣在做什么？"
- "周末你想宅在家但你的伴侣想出去社交，你们会怎么处理？"

### 矛盾检测线索
- 声称"什么都行随便"但在具体场景中有强烈偏好 → 压抑了真实需求
- 理想一天的描述中完全没有伴侣出现 → 可能对关系的融入有疑虑

## 维度五：核心价值观与人生信念

### 核心目标
挖掘 **不可协商的核心信念**，这些信念决定了长期关系的根基。

### 核心问题

1. **你觉得什么是"好的关系"？能描述一下你心中理想的关系状态吗？**
   - 目的：核心问题，直接映射关系期待
   - 追问方向：这个标准从哪来的？身边有这样的榜样吗？

2. **在一段关系中，你觉得最不能接受的事情是什么？你的"底线"是什么？**
   - 目的：底线往往指向最深层的恐惧和最核心的需求
   - 追问方向：这个底线是从什么时候开始有的？跟某段经历有关吗？

3. **你怎么看待婚姻？你觉得结婚对你来说意味着什么？**
   - 目的：对婚姻的态度反映安全感、承诺感、社会认同等多层需求
   - 追问方向：你身边结婚的人，你羡慕谁的状态？

4. **当你和伴侣意见不一致时，你通常会怎么处理？**
   - 目的：冲突处理风格是关系健康度的核心指标
   - 追问方向：你希望伴侣怎么处理冲突？

5. **如果要你在事业和关系之间排优先级，你会怎么排？**
   - 目的：识别价值优先级和潜在的角色冲突

### 情境假设题

- "如果你的伴侣得到一个很好的机会但需要去另一个城市两年，你怎么看？"
- "如果你发现伴侣隐瞒了一件不太严重的事，你会怎么反应？"
- "你和伴侣在要不要孩子的问题上观点不同，你会怎么处理？"

### 矛盾检测线索
- 说"事业和关系一样重要"但具体场景中总是牺牲关系 → 实际优先级与表述不符
- 对冲突的描述只有对方的问题 → 可能缺乏在关系中自我觉察的能力
- 底线过多 → 可能是过度防御，需要区分真正的核心底线和恐惧驱动的防护墙

## 维度六：未来规划与关系期待

### 核心目标
了解用户 **对未来的具体想象**，判断择偶偏好是否服务于真实的人生方向。

### 核心问题

1. **五年后你希望自己的生活是什么样的？十年后呢？**
   - 目的：人生方向决定了需要什么样的伴侣
   - 追问方向：在这个画面里，你的伴侣在做什么？

2. **你对未来的家庭有什么想象？（住在哪、生活方式、家庭氛围等）**
   - 目的：具体想象比抽象描述更能揭示真实需求
   - 追问方向：这个想象跟你成长的家庭像不像？

3. **你怎么看待养育下一代这件事？**
   - 目的：对子女的态度反映责任观、价值传承和对关系稳定性的期待
   - 追问方向：如果伴侣的想法跟你不一样呢？

4. **有什么是你一定想要在有生之年做到的？这件事需要伴侣的支持吗？**
   - 目的：识别个人使命和伴侣在人生蓝图中的位置

5. **你觉得"在一起变老"是什么感觉？这个想象让你兴奋还是焦虑？**
   - 目的：对长期承诺的深层情感反应

### 情境假设题

- "如果你现在的生活方式需要做一个大的改变才能拥有理想的关系，你愿意改变什么？不愿意改变什么？"
- "假设你70岁回看自己的感情生活，你最希望看到的画面是什么？"

### 矛盾检测线索
- 未来规划中伴侣角色模糊 → 可能尚未真正准备好进入关系
- 对未来的想象过于完美和具体 → 可能存在理想化偏差
- 对变老和长期承诺有明显焦虑 → 可能有回避型依恋特征

## 综合分析指引

### 交叉验证原则

问询过程中需要在不同维度之间交叉验证——**每一对交叉验证都是潜在的矛盾发现点**：

| 交叉验证对 | 验证内容 | 典型矛盾信号 |
|-----------|---------|-------------|
| 基本信息 × 择偶偏好 | 用户的条件是否与自身硬件条件校准合理 | 自身条件一般但择偶标准极高（或反之） |
| 关系定位 × 择偶偏好 | 表述的条件是否服务于其真实的关系定位 | 说找"爱人"但条件清单全是硬件指标 |
| 关系定位 × 感情经历 | 过去最幸福/最遗憾的部分是否与声称的定位一致 | 说找"搭子"但最遗憾的全是"失去心动感" |
| 自我认知 × 感情经历 | 用户的自我描述是否与关系中的行为一致 | 说自己"很独立"但每段关系都重度依赖 |
| 原生家庭 × 择偶偏好 | 偏好是否在重复或对抗原生家庭模式 | 父亲沉默→极度要求"会沟通"（对抗型补偿） |
| 价值观 × 生活方式 | 声称的价值观是否体现在日常选择中 | 说"重视健康"但生活方式完全不健康 |
| 感情经历 × 未来规划 | 从过去的教训是否真正影响了未来方向 | 说"学到了教训"但择偶标准完全没变 |
| 自我认知 × 依恋风格 | 用户的自我评价与依恋行为模式是否匹配 | 说自己"安全型"但行为明显焦虑/回避 |
| 认知功能 × 沟通偏好 | 用户要求的沟通方式是否与自身认知风格匹配 | 自己是慢热深思型但要求对方"马上沟通" |
| 偏好强度 × 情绪强度 | 声称"必须"的条件谈论时情绪平淡，"加分项"却激动 | 真正的核心需求可能被错误标注了优先级 |

### 信息充分性检查

在进入确认阶段前，确保以下关键信息已获取：

- [ ] 用户的基本硬件信息（年龄、学历、职业、经济水平、定居城市）
- [ ] 用户的关系定位（爱人/搭子/队友，主+辅）
- [ ] 用户的自我认知和自我接纳程度
- [ ] 至少一段感情经历的详细叙述（或明确了无经历的原因）
- [ ] 原生家庭关系的基本图景
- [ ] 日常生活方式和社交偏好
- [ ] 对关系的核心期待和底线
- [ ] 对未来生活的基本想象
- [ ] 初步的认知功能偏好判断（至少 2-3 个功能轴的倾向）
- [ ] 至少 2-3 个矛盾点或值得深挖的线索

### 矛盾收敛性检查

在进入确认阶段前，检查矛盾追踪表：

- [ ] 关系定位矛盾（如存在）已追问至收敛
- [ ] 核心需求矛盾（如存在）已追问至收敛或标记为开放性矛盾
